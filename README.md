# Описание проекта
Данный проект содержит набор автоматизированных тестов для системы Web Automation Torture Lab. Тесты написаны на языке Python с использованием фреймворка Selenium WebDriver.

# Технологический стек
Язык: Python 3.10+, Фреймворк тестирования: Pytest, Инструмент автоматизации: Selenium, Отчетность: Allure, Архитектура	Page Object Model (POM)

# Запуск проекта
1. Подготовка окружения
Склонируйте репозиторий и установите необходимые зависимости (см. файл requirements)

2. Настройка конфигурации
Проект использует файл config.py для хранения базовых настроек.

3. Варианты запуска тестов:
- Запуск всех тестов - pytest
- Запуск конкретного тестового файла - pytest tests/test_login.py
- Запуск конкретного теста по имени - pytest -k "test_login_success"
- Запуск с подробным выводом - pytest -v
- Запуск с остановкой при первой ошибке  - pytest -x

# Отчетность
Отчетность на проекте реализована с помощью Allure.

Соответствующие команды:
- Запуск тестов с сохранением результатов - pytest --alluredir=allure-results
- Генерация и открытие отчета - allure serve allure-results
- Генерация статистического отчета - allure generate allure-results -o allure-report
allure open allure-report

В отчете доступны:
- Пошаговое выполнение тестов
- Время выполнения каждого теста
- Параметры тестов

# Структура проекта
- pages/ - Page Object классы. Содержат локаторы элементов и методы работы с ними.
- test_data/ - Хранилище тестовых данных.
- tests/ - Набор тестов. Каждый файл тестирует конкретную функциональную область. Включает в себя подразделы API и UI
- config.py	- Настройки окружения.
- conftest.py - Глобальные фикстуры для pytest.
- pytest.ini	Конфигурация pytest.

# Архитектура проекта (Page Object Model)
Проект построен на паттерне Page Object Model.

- BasePage (pages/base_page.py)
BasePage является родительским классом для всех Page Object'ов. Содержит общие методы-обертки необходимые для работы (например, send_keys(locator, value), assert_element_visible(locator), open() и т.д.)
Все методы автоматически логируются в Allure через декоратор @allure.step для наглядности.

- Page Object'ы
Каждая страница (AdminPage, BoardsPage, LoginPage и т.д.) наследуется от BasePage и хранит локаторы элементов внутри класса и содержит необходимые методы

- Тестовые данные (test_data/)
Модуль test_data/users.py содержит объекты пользователей с различными ролями и правами

-Конфигурация (config.py)
Файл config.py содержит базовые настройки проекта

- Тестовый слой (tests/)

- UI-тесты (tests/ui/)
Проверяют пользовательские сценарии через браузер
- API-тесты (tests/api/)
Проверяют серверную логику

# Запуск с Docker
Тесты с проекта возможно запускать в Docker-контейнере.

1. Подготовка
Убедитесь, что:
- Тестируемое приложение доступно по адресу http://localhost:3000
- приложение Docker установлено и запущено

2. Сборка образа
docker build -t qa-tests-finals

3. Запуск тестов
- Запуск всех тестов
docker run --rm --network host qa-tests-finals

- Запуск конкретного теста
docker run --rm --network host qa-tests-finals pytest tests/test_*.py -v

- Открыть: http://localhost:5050
